## 1. Brand Foundation

### What is PRP?
PRP is a personal operating system built around the four resources that define human life:

> **Money · Time · Energy · Health**

It is not a productivity app. It is not a finance app. It is a unified system for people who treat their life with the seriousness of a business — tracking, planning, and optimizing across all four dimensions simultaneously.

### Positioning
- **Target**: Ambitious individuals who want a single source of truth for their life's resources
- **Stage**: Alpha (2026) → Beta → Product Hunt launch
- **Tone**: Precise. Intentional. Disciplined. No fluff.
- **Anti-tone**: Gamified, cutesy, over-motivational, generic productivity

### The Four Pillars

| Pillar | Emoji | Color | Purpose |
|--------|-------|-------|---------|
| Money | 💰 | Teal `#3AB8A8` | Accounts, investments, debts, transactions |
| Time | ⏱️ | Blue `#6A8EF0` | Schedule blocks, calendar, task manager |
| Energy | ⚡ | Purple `#AA70EE` | Focus sessions, fasting, goals |
| Health | ❤️ | Orange `#D07848` | Habits, streaks, body metrics |

## 2. Logo & Wordmark

### Primary Wordmark
```
PRP
```
- **Font**: PlayfairDisplay — Black (900 weight)
- **Letter-spacing**: -0.5px to -1px
- **Color**: `#E8E8E8` on dark / `#0F172A` on light

### Logo Mark (Icon)
A square with 12px rounded corners containing the letter **"P"** in white/black:
- **Dark version**: `#22C55E` fill, `#0A0A0A` letter
- **Light version**: `#22C55E` fill, white letter
- **Monochrome dark**: `#2A2A2A` fill, `#E8E8E8` letter
- **Minimum size**: 24×24px (digital), 10mm (print)

### Full Lockup
```
[P]  PRP
     Personal Resource Planner
```
- Logo mark left, wordmark + tagline right
- Tagline: 11px, ALL CAPS, 2.5px letter-spacing, `#888888`

### Tagline
> "Money · Time · Energy · Health"

The middle-dot separator (·) is always used between pillars, never dashes or commas.

### Clear Space
Minimum clear space = 1× the width of the "P" logo mark on all sides.

### Don'ts
- ❌ Don't stretch or distort the logo
- ❌ Don't use the purple/indigo from old templates — that's not PRP
- ❌ Don't place the green logo on a green background
- ❌ Don't use gradients on the wordmark itself
- ❌ Don't add drop shadows to the logo

- **One accent per screen** — don't mix green + gold in the same prominent position

- **Red is for danger** — only destructive actions (delete, reset)
- **Orange is for warning** — security alerts only
- **Category colors are decorative** — never use them as primary brand colors

## 4. Typography

### Type Scale

| Role | Family | Size | Weight | Usage |
|------|--------|------|--------|-------|
| Display | PlayfairDisplay | 40px | 900 (Black) | Hero headings, splash screens |
| Heading L | Poppins | 24px | 700 (Bold) | Screen titles |
| Heading M | Poppins | 20px | 700 (Bold) | Section headers |
| Heading S | Poppins | 18px | 600 (SemiBold) | Card titles |
| Title | Poppins | 16px | 600 (SemiBold) | List item headers |
| Body L | Roboto | 14px | 400 (Regular) | Main body text, line-height 1.6 |
| Body M | Roboto | 12px | 400 (Regular) | Secondary body, line-height 1.5 |
| Body S | Roboto | 11px | 400 (Regular) | Captions, metadata |
| Label L | System | 12px | 600 (SemiBold) | Button text, tab labels |
| Label M | System | 10px | 500 (Medium) | Chip labels, tags |
| Label S | System | 9px | 400 (Regular) | Navigation labels, ALL CAPS |

### Font Loading (Web)
```
PlayfairDisplay — google fonts / local assets (Display headings)
Poppins         — google fonts / local assets (UI headings)
Roboto          — system fallback / local assets (body)
```
System fallback stack: `-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif`

### Typography Rules
- **PlayfairDisplay** = emotional weight — hero text, auth screens, dialog titles
- **Poppins** = structural — navigation, section headers, card titles
- **Roboto** = functional — body, tables, data
- **Never** use monospace (IBMPlexMono has been fully removed from the codebase)
- Letter-spacing on ALL CAPS labels: 1.5–2.5px always
- Line-height for body: 1.6–1.7; headings: 1.2–1.3

## 6. UI Components

### Buttons

**Primary (CTA)**
- Background: `#22C55E`
- Text: `#0A0A0A` (black on green)
- Font: 12–14px, SemiBold (600), letter-spacing 0.3–0.5px
- Height: 44px minimum (accessibility)
- Border radius: 8px
- Hover: slight darken / scale(1.01)
- Disabled: 40% opacity

**Destructive**
- Background: `#E05050`
- Text: `#FFFFFF`
- Same dimensions as primary

**Secondary/Outlined**
- Background: transparent
- Border: 1px solid `#2A2A2A` (dark) / `#E2E8F0` (light)
- Text: `#888888` → `#E8E8E8` on hover
- Height: 44px minimum

**Ghost/Text**
- Background: transparent
- Text: `#22C55E`
- No border

### Cards
- Background: `#1A1A1A`
- Border: 1px solid `#2A2A2A`
- Border radius: 12px
- Elevation: 0 (flat design — no shadows)
- Hover: background → `#222222`
- Padding: 12px (compact) / 16px (standard)

### Inputs / Text Fields
- Background: `#111111`
- Border: 1px solid `#2A2A2A`
- Border radius: 8px
- Focus border: 1.5px solid `#22C55E`
- Label: 12px, `#888888`
- Error border: `#E05050`

### Badges / Tags
- Border radius: 4px (rectangular) or 20px (pill)
- Padding: 3–4px vertical, 8–12px horizontal
- Font: 9–11px, SemiBold, letter-spacing 1px+
- **Alpha badge**: gold `#C8A050` text, gold 12% bg + border
- **Category tag**: category color text + 8% opacity bg

### Navigation (Bottom bar — mobile)
- Background: `#111111`
- Active icon + label: `#22C55E`
- Inactive: `#888888`
- Active indicator: `#22C55E` at 15% opacity
- Tab height: 64px, icon size: 20px, label: 9px

### Navigation (Sidebar — desktop)
- Background: `#111111` with left border `#2A2A2A`
- Logo at top with gold accent
- Active item: green left border + `#22C55E` icon/text
- Section dividers: `#2A2A2A`

## 8. Motion & Animation

### Principles
- **Purposeful** — every animation communicates meaning
- **Fast** — 150–300ms for UI feedback, never longer than 400ms
- **Smooth** — ease-in-out curves, no linear

### Duration Scale
| Speed | Duration | Usage |
|-------|----------|-------|
| Instant | 100ms | Button press feedback, tap highlights |
| Fast | 150ms | Icon transitions, small state changes |
| Normal | 250ms | Card entrance, drawer open/close |
| Slow | 350ms | Modal appear, page transitions |

### Curves
- Default: `Curves.easeInOut`
- Entrance: `Curves.easeOut`
- Exit: `Curves.easeIn`
- Bounce: `Curves.elasticOut` (for completion celebrations only)

## 9. Voice & Tone

### Writing Principles

1. **Direct** — say exactly what you mean, no padding
2. **Precise** — use exact numbers, real data
3. **Calm** — no urgency theater (avoid "URGENT!", "Act now!")
4. **Respectful** — treat users as adults managing serious goals

### Notification Copy Examples

✅ Good:
- "Fajr block starts in 5 minutes"
- "Habit streak: 7 days 🔥"
- "All 6 habits completed today"

❌ Bad:
- "Don't forget to stay on track today! You've got this! 💪🎉"
- "REMINDER: You have habits to complete!"

### Error Messages
- State what happened, not blame
- Offer a path forward
- Short: under 40 words

Example:
- ❌ "An error has occurred. Please try again later."
- ✅ "Couldn't save. Check your connection and try again."

### Button Labels
- Always verb-first: "Save changes", "Delete habit", "Add block"
- Never vague: ❌ "OK", "Continue", "Submit" → ✅ "Confirm", "Save", "Add"

### Email Subject Lines
```
Confirm your PRP account
Your PRP sign-in link
Reset your PRP password
PRP: Password changed
Your PRP invitation
```
